# ApexCore

Кроссплатформенный бенчмарк системы для **Windows 11** и **Astra Linux**.
Считает производительность компьютера на основе **Roofline-модели** — балл
показывает, какой процент от теоретического пика собственного железа реально
использует ваш ПК. Эталонный компьютер не нужен: баллы напрямую сравнимы
между разными системами.

## Что умеет ApexCore

1. **Общая оценка производительности системы** — короткий композитный прогон
   CPU + RAM + загрузочного диска. На выходе одно число «попугаев» в шкале до
   10 000 (4000–6000 типовой десктоп, 6000–8000 — мощная конфигурация, 10 000 —
   теоретический потолок). Объединяет: DGEMM-нагрузку на CPU+RAM,
   STREAM-пропускную способность памяти и три диск-теста — последовательное
   чтение, случайное чтение, последовательную запись. Длительность ~1.5 минуты.
2. **Стресс-нагрузка** — длительный параллельный стресс CPU + RAM с термальной
   защитой (watchdog) и метрикой Frame-Rate-Stability % (по образцу UL 3DMark,
   pass / fail @ 97 %). Свой детерминированный «Стресс-балл» в той же шкале
   ×10 000, его можно сравнивать «бок о бок» с общей оценкой — разница
   показывает, насколько cooling/throttling сажает мощность под нагрузкой.
3. **Детальная общая оценка (scoring v2)** — 12 микробенчмарков в 5 категориях
   (Memory / FLOPS / Integer / Crypto / Fractals), итоговый балл по
   Roofline-модели в шкале ×1000. Подходит для покомпонентной диагностики
   и для CI: пресеты `fast` / `standard` / `accurate` с 95 % CI на лог-шкале.
4. **Расширенный тест ОЗУ и кеша (Ram & Cache)** — диагностическая таблица
   4×4 со скоростями Read / Write / Copy / Latency для DRAM и L1 / L2 / L3
   кеша. Работает на NumPy с JIT через numba.
5. **Аналог Windows Winsat** — собственная гибридная реализация со шкалой
   1.0–9.9 (CPUScore, MemoryScore, DiskScore, GraphicsScore, WinSPRLevel).
   CPU / Memory / Disk считаются своими движками; графика и D3D — через
   нативный `winsat dwm`. Только Windows.

Дополнительно: раздел **«Датчики»** — live-дашборд температур, частот,
напряжений, мощности, вентиляторов и накопителей с группировкой по
карточкам CPU / GPU / Вентиляторы / Диски / Материнская плата,
sparkline-история, focus-режимы и индикатор троттлинга. На Windows
поддерживается несколько источников показаний: HWiNFO / CoreTemp /
AIDA64 (через Shared Memory без admin), LibreHardwareMonitor (через
встроенную DLL), AMD Ryzen Master, ACPI, WMI — с автоматическим
fallback'ом, если основной источник недоступен (Memory Integrity, Smart
App Control, антивирус). Полная история прогонов в SQLite, экспорт в
JSON / CSV. Опциональный **Web UI** (FastAPI) дублирует все разделы в браузере.

## Установка

Релиз **v1.0.0** — единый: под одним номером лежат установщики на обе ОС.
Ядро (методика scoring, движки, CLI) кроссплатформенное; различаются только
адаптеры ОС и упаковка. Скачать — на странице
[Releases](https://github.com/Lelbry/ApexCore/releases).

### Windows 11 — установщик

Скачайте **`apexcore-setup-win-1.0.0.exe`** и запустите. Мастер с графическим
интерфейсом (WebView2) поставит ApexCore, добавит в `PATH` и опционально
зарегистрирует сервис `apexcore_sensord` — после этого чтение CPU-температуры /
Vcore / потребления работает **без UAC** при каждом запуске. Один UAC при
установке покрывает всё. Дальше — `apexcore` из любой консоли.

### Astra Linux — пакет .deb

```bash
sudo apt install ./apexcore_1.0.0_amd64.deb
```

Пакет автономный: внутрь вшит relocatable CPython (`/opt/apexcore/.venv`),
поэтому установка не зависит от системного `python3` и его версии. Первый
запуск открывает браузерный wizard для настройки.

> Тестовый стенд Astra: **SE 1.8.5.46** «Орёл защищённости» (бюллетень
> № 2026-0224SE18, ядро 6.1.158, AMD Ryzen 7 6800H + Radeon 680M).

### Из исходников (Windows 11 / Astra Linux / Ubuntu)

```bash
git clone https://github.com/Lelbry/ApexCore.git
cd ApexCore
python -m venv .venv
# Windows:  .\.venv\Scripts\Activate.ps1   (один раз: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned)
# Linux:    source .venv/bin/activate
pip install -e ".[dev,fast]"
# опционально: [webui] — веб-интерфейс, [windows] — расширенная телеметрия WMI/pywin32
sudo apt install stress-ng sysbench lm-sensors   # Linux, опц. внешние стресс-движки
```

DLL-и LibreHardwareMonitor (~3 МБ) идут в репозитории и подгружаются
автоматически — отдельный процесс LibreHardwareMonitor не нужен. При первом
запуске `apexcore` может попросить UAC, чтобы зарегистрировать kernel-сервис
чтения сенсоров (один раз). Если CPU-температура не считывается,
`apexcore doctor` покажет конкретную причину (HVCI, Smart App Control,
Defender) и предложит решение.

> <a name="windows-server"></a>**Windows Server.**¹ По зависимостям ApexCore
> совместим с Windows Server (2016 / 2019 / 2022): ядро (CLI, scoring,
> микробенчмарки, общая оценка, стресс, Ram & Cache) — это чистый Python +
> numpy / scipy / psutil без привязки к клиентским компонентам Windows 11, а
> Windows-extras (`pywin32`, `wmi`, `pythonnet`) объявлены под
> `sys_platform == 'win32'`, что покрывает и Server. Жёсткой проверки
> «только Windows 11» в коде нет (определение ОС идёт через `platform.release()`
> лишь для отображения). Ограничения: графический подскор аналога WinSAT
> (`winsat dwm`) требует компонента «Возможности рабочего стола» (Desktop
> Experience) — на Server Core он деградирует в «нет данных», остальные
> подскоры (CPU / Memory / Disk) считаются своими движками и работают;
> WebView2-установщик требует WebView2 Runtime (на Win 11 предустановлен, на
> Server ставится вручную, ~2 МБ) — установка из исходников этого не требует;
> драйверы сенсоров (PawnIO / LHM) подчиняются той же политике подписи
> драйверов, что и на клиенте.
>
> ¹ ⚠ На реальной Windows Server система **не разворачивалась** — вывод сделан
> по анализу зависимостей и кода, не по факту установки на саму ОС.

## Быстрый старт

```bash
apexcore                                       # интерактивное меню (рекомендуется)
apexcore info                                  # сведения о системе
apexcore webui                                 # локальный Web UI на 127.0.0.1:8765
apexcore sensors                               # раздел «Датчики»: live-дашборд
apexcore micro run --preset standard           # детальный балл ~15 мин
apexcore micro run --preset fast               # быстрый балл ~5 мин
apexcore ram-cache run                         # тест ОЗУ и кеша 4×4 (~2 мин)
apexcore winsat run                            # аналог Win32_Winsat, шкала 1.0–9.9 (Windows)
apexcore runs list                             # история прогонов
apexcore doctor                                # диагностика сенсоров
```

В TUI-меню раздел **«Общая оценка производительности системы»** (пункт 4)
запускает комплексный бенчмарк и показывает крупную плашку «Ваш итоговый
балл» с цветной таблицей по подсистемам.

## Что значит балл

Все балльные шкалы ApexCore идейно одинаковы: `измеренное / теоретический
пик железа`. В отличие от Geekbench / SPEC, **не нужен «эталонный ПК»** —
балл интерпретируется напрямую как процент эффективности железа.

| Балл общей оценки (×10 000) | Что означает |
|---|---|
| **10 000** | теоретический потолок архитектуры (на реальном железе обычно недостижим) |
| **7000–10 000** | мощная актуальная конфигурация |
| **5000–7000** | хороший современный десктоп |
| **3500–5000** | средний десктоп / рабочая станция |
| **< 3500** | слабая, устаревшая или виртуальная среда |

| Балл детальной общей оценки (×1000) | Что означает |
|---|---|
| **1000** | физический максимум архитектуры (на практике недостижим) |
| **600+** | очень хорошо — система работает близко к своему пику |
| **400–600** | типично для современного железа на реальных нагрузках |
| **200–400** | средне — фоновые процессы или лёгкое термоограничение |
| **< 200** | серьёзное ограничение (троттлинг, батарейный режим, фон) |

Шкалы намеренно разные («попугаи» ×10 000 для общей оценки удобнее запомнить
как 4-значное число, ×1000 для детальной — как у BAPCo SYSmark). Принцип
расчёта общий: геометрическое среднее безразмерных ratio.

> **Поддержка любого x86-64 CPU.** Балл не «зашит» под конкретный процессор:
> Roofline-пики считаются из числа ядер, частоты (`psutil` / парсинг модели) и
> SIMD-уровня (эвристика по строке модели — Intel-поколения, AMD Zen). Для
> любого нераспознанного x86-64 берётся безопасный fallback (`sse4`, гомогенная
> формула `ядра × ops/такт × частота`), а где аналитического потолка нет —
> empirical-proxy. Гибридные Intel (Alder / Raptor Lake, P+E split) уточняются
> по таблице SKU, но её отсутствие для нового CPU лишь немного завышает пик, не
> ломает расчёт. ARM / RISC-V в v1.0 вне области Roofline.

## Пресеты точности (для детальной оценки)

| Пресет | Прогонов | Доверительный интервал | Время |
|---|---|---|---|
| `fast` | 1 | — | ~5 мин |
| `standard` (рекомендуется) | 3 | median-of-3 | ~15 мин |
| `accurate` | 5 | 95 % CI на лог-шкале (Lilja 2000) | ~25 мин |

## Структура репозитория

```
src/apexcore/       домен / use-cases / адаптеры Win+Linux / CLI / Web UI
tests/              pytest unit + integration (~1090)
scripts/            сборка дистрибутивов и валидация
packaging/          Inno Setup + WebView2 bootstrapper (Windows), debian (Astra)
docs/               методика scoring, стресс-тест, troubleshooting
```

## Тесты

```bash
pytest -q          # ~1090 unit + integration
ruff check .       # линт
mypy src           # типы
```

## Сборка дистрибутивов

- **Windows** — `scripts/build_windows.ps1` → `apexcore-setup-1.0.0.exe`
  (PyInstaller + Inno Setup + WebView2 bootstrapper). Требует .NET 8 SDK,
  Inno Setup 6, ImageMagick.
- **Astra Linux** — `scripts/build_astra.sh` → `apexcore_1.0.0_amd64.deb`
  (debhelper + dpkg-buildpackage).

## Требования

- Python **≥ 3.10** (рекомендуется 3.11.9) — только для установки из исходников;
  установщики несут собственный runtime.
- Windows 11 или Astra Linux / Ubuntu 22.04+. См. сноску про
  [Windows Server](#windows-server).
