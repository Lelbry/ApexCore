# ApexCore 1.0.0 — релиз под Astra Linux

**Дата:** 27.05.2026 · **Пакет:** `apexcore_1.0.0_amd64.deb` (~122 МБ)
**Проверено на:** Astra Linux SE 1.8.5.46 (бюллетень № 2026-0224SE18, ядро 6.1.158;
AMD Ryzen 7 6800H + Radeon 680M iGPU, Phison NVMe).

Первый стабильный релиз под Astra Linux. Главное — пакет стал **полностью
автономным**: больше не зависит от системного `python3`, ставится «из коробки».

---

## Что нового и что починили

- **Автономная установка.** Внутрь `.deb` вшит relocatable CPython
  (python-build-standalone) как `/opt/apexcore/.venv`. Установка не зависит
  от версии системного Python — пакет одинаково ставится на разные сборки Astra.
- **Мастер первого запуска (в браузере):** добавлен экран завершения после
  кнопки «Завершить»; запрашиваются корректные Linux-права
  (`dmidecode → cap_dac_read_search`, без лишней эскалации `smartctl`).
- **Общая оценка системы:** диск-бенч пишет в `~/.cache/apexcore-bench`, если
  корень недоступен на запись → итоговый балл и данные по диску теперь считаются.
- **Стресс-тест:** встроенные движки DGEMM + STREAM дают сопоставимые GFLOPS/ГБ-с
  → отдельный итоговый балл стресса выводится корректно.
- **Память:** DGEMM в 1 поток + `threadpoolctl`, STREAM = `logical/4` потоков →
  больше нет нехватки ОЗУ и краша браузера на «Общей оценке».
- **Web UI:** лента прогонов и `/api/trend` устойчивы к битой БД
  (`SQLITE_CORRUPT` больше не роняет страницу — возвращается пустой тренд).
- **Реальные данные железа** в WebUI (boot-диск, объём/тип/частота ОЗУ) вместо
  заглушек; убран устаревший единый «итоговый балл» микро-теста.

---

## Системные требования

- Astra Linux SE 1.8.x (Debian 12 base) или совместимая, x86-64.
- Зависимости ставятся автоматически из репозитория Astra:
  `stress-ng`, `sysbench`, `lm-sensors`, `smartmontools`, `pciutils`,
  `dmidecode`, `util-linux`, `x11-utils`, `libcap2-bin`, `policykit-1 | polkit`.

## Установка

Перейдите в каталог с файлом `apexcore_1.0.0_amd64.deb` и выполните:

```bash
# Установка (apt сам подтянет зависимости).
# Префикс ./ обязателен — иначе apt ищет пакет в репозитории, а не файл.
sudo apt install ./apexcore_1.0.0_amd64.deb
```

Обновление поверх предыдущей версии — та же команда (apt сделает upgrade):

```bash
sudo apt install ./apexcore_1.0.0_amd64.deb
```

> Если apt ругается предупреждением вида «_apt … sandbox … permission denied»
> при чтении файла из домашнего каталога — скопируйте `.deb` в `/tmp` и ставьте
> оттуда:
> ```bash
> cp apexcore_1.0.0_amd64.deb /tmp/ && sudo apt install -y /tmp/apexcore_1.0.0_amd64.deb
> ```

## Первый запуск — мастер настройки

```bash
apexcore setup                 # откроет мастер в браузере (рекомендуется)
# или, если браузер не открылся автоматически:
apexcore setup --no-browser    # выведет URL — откройте его в браузере вручную
```

Мастер запросит права на чтение сенсоров (через `pkexec`) и подготовит систему.

---

## Проверка после установки

```bash
apexcore --version             # apexcore 1.0.0
apexcore info                  # системная информация + строка прав (capabilities)
apexcore doctor                # диагностика сенсоров (CPU/GPU/диск + здоровье БД)
apexcore webui --port 8765     # Web UI на http://127.0.0.1:8765
apexcore micro run --preset fast   # короткий прогон оценки (~30 сек)
```

## Удаление

```bash
sudo apt remove apexcore          # удалить пакет (конфиги оставить)
sudo apt purge apexcore           # удалить пакет вместе с конфигами
```
