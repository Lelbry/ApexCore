# НИР — научно-исследовательские материалы ApexCore

В этой папке собираются **аналитические документы**, обосновывающие методические
решения проекта: почему мы выбрали ту или иную метрику, на каких научных
источниках стоит формула, чем наш подход отличается от существующих систем.

В отличие от `docs/` (где собрана инженерная документация — архитектура,
API, troubleshooting), документы в `docs/НИР` ориентированы на **защиту
методологии** перед научной или экспертной аудиторией: ссылки на peer-reviewed
работы, эмпирические замеры, таблицы сравнения с эталонами.

## Документы

| Файл | Тема |
|---|---|
| [`winsat_methodology_comparison.md`](winsat_methodology_comparison.md) | Сравнение методики ApexCore «Наследие Winsat» (CPU и Memory подскоры) с native Microsoft Winsat. Обоснование выбора собственных бенчмарков (AES-256 + SHA-1 для CPU, cache-aware memory_read для RAM) и переиспользования native winsat для Disk/Graphics/D3D. |

## Шаблон для новых документов

Каждый документ НИР должен содержать:

1. **Резюме** (1–2 абзаца) — основной тезис.
2. **Контекст и мотивация** — что побудило к работе, какая проблема решается.
3. **Методология** — что именно мы делаем, формулы, конкретные ссылки на код.
4. **Эмпирическое сравнение** — таблица замеров на эталонной машине,
   сопоставление с альтернативами.
5. **Научное обоснование** — почему наш выбор согласуется с peer-reviewed
   источниками и индустриальной практикой.
6. **Ограничения и future work** — где наш подход слабее, что планируется
   улучшить.
7. **Источники** — список с DOI/URL.

Документы пишутся на русском, формулы и термины — стандартные академические
(когда возможно — со ссылкой на исходную публикацию).
