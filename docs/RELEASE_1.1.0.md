# ApexCore 1.1.0 — кроссвендорный модуль тестирования GPU

**Дата:** 04.07.2026 · **Пакеты:** `apexcore-setup-1.1.0.exe` (Windows) ·
`apexcore_1.1.0_amd64.deb` (Astra Linux)

Главное в релизе — **отдельный модуль оценки видеокарты**: свой Roofline-бенчмарк
(вычисления + видеопамять + шина) со сквозным баллом и GPU-стресс на
термостабильность. Кроссвендорный путь через OpenCL, без единой новой
Python-зависимости. Заодно сняты backward-compat шимы `benchkit → apexcore`,
которые держались один релиз.

---

## Что нового

### GPU-бенчмарк (Roofline, шкала ×10 000)

Одно число вроде `6500` — насколько близко карта работает к своему
**архитектурному пику** по двум осям: вычисления (FP32) и пропускная способность
видеопамяти (VRAM). Методика повторяет CPU-Roofline и общую оценку: знаменатель —
теоретический предел *конкретной* карты (число вычислительных блоков × частота ×
операций за такт), а не «попугаи как у соседа».

- **Измеряемые нагрузки:** FP32 и FP64 (GFLOPS), пропускная способность VRAM
  (STREAM-triad, GB/s), скорость шины PCIe в обе стороны (host→device и
  device→host, GB/s).
- **Итоговый балл** = `GM(r_fp32, r_mem) × 10 000`, где `r_*` — доля от
  архитектурного пика (после clamp ≤ 1.0, чтобы заводской разгон/буст не задирал
  балл выше 10 000).
- **FP64 и PCIe — информационные, в балл не входят.** На игровых GeForce
  (FP64 = 1/64) и встроенных Intel/AMD iGPU двойная точность намеренно урезана —
  её включение несправедливо занизило бы игровую карту; PCIe же характеризует
  шину/платформу, а не саму карту. Оба показателя измеряются и выводятся сырыми
  скоростями.
- **Та же шкала ×10 000**, что у общей оценки и стресс-балла — все метрики
  сравниваются «бок о бок».

### GPU-стресс (термостабильность)

Кроссвендорный аналог FurMark на compute-нагрузке, headless и без внешних утилит.

- Длительная максимальная FP32-нагрузка («power virus») выбранное время +
  посекундная телеметрия: температура, мощность, частота ядра, загрузка GPU.
- Итог — **вердикт `PASS / WARN / FAIL / UNKNOWN`**: небольшой boost-settle
  трактуется максимум как WARN, а FAIL — только достижение теплового лимита или
  крупный обвал частоты. `UNKNOWN` — когда телеметрия недоступна (нагрузку
  прогнали, но судить о стабильности не по чему).
- Спарклайн температуры + сводка (max/avg/min) в отчёте.

### CLI, меню и Web UI

- **CLI:** `apexcore gpu list` (обнаруженные устройства), `apexcore gpu run`
  (полный бенчмарк + балл + сохранение в БД), `apexcore gpu test`
  (одиночный замер одной нагрузки без скоринга), `apexcore gpu stress`
  (термотест с вердиктом).
- **Меню:** новый пункт «Оценка производительности GPU» на главном экране.
- **Web UI:** отдельный экран с двумя режимами (бенчмарк + стресс), выбором
  устройства, прогрессом и карточкой результата.
- **Единая История:** прогоны GPU-бенчмарка и GPU-стресса встали в общую ленту
  «История ваших тестов» рядом со stress / micro / winsat / общей оценкой.

### Хранилище

- **Схема БД v6** (additive-миграция, ничего не дропается): добавлены таблицы
  `gpu_benchmark_runs` (v5) и `gpu_stress_runs` (v6). Всего 6 таблиц прогонов.

### Снос benchkit-шимов

Совместимость со старыми установками `benchkit` (до переименования в
`apexcore`) держалась один релиз и теперь **удалена**: CLI-alias `benchkit`,
автотрансляция ENV `BENCHKIT_* → APEXCORE_*`, миграция старых data-dir
(`~/.local/share/benchkit/`), `/usr/bin/benchkit` symlink и
`Conflicts/Replaces/Provides: benchkit` в `.deb`. `apt install apexcore` поверх
старого `benchkit` больше не поддерживается атомарно.

## Платформы

| Платформа | Статус GPU-модуля |
|---|---|
| **Windows** | **Проверено** на NVIDIA RTX 4070 Ti + Intel UHD 770 (дискретная + встроенная). Полный цикл: перечисление, бенчмарк, стресс, сохранение, WebUI. |
| **Astra Linux / AMD** | **Graceful degrade.** Без OpenCL ICD модуль показывает «GPU/OpenCL не обнаружен» и не мешает остальному apexcore. Полноценный GPU-compute на **Radeon 680M** ждёт прогона на стенде и требует установленного OpenCL ICD. |

> На Astra `.deb` OpenCL-loader и универсальный ICD добавлены как
> **`Recommends`** (`ocl-icd-libopencl1`, `mesa-opencl-icd`) — они не блокируют
> установку, но дают модулю шанс запуститься, если apt их подтянет. На
> дискретных картах свежий проприетарный драйвер приносит собственный OpenCL
> runtime.

## Проверка после установки

```bash
apexcore --version                 # apexcore 1.1.0
apexcore gpu list                  # обнаруженные GPU (или «не обнаружен»)
apexcore gpu run -d 0              # полный бенчмарк устройства №0 → балл ×10 000
apexcore gpu test -d 0 -w fp32     # одиночный замер FP32 (сырые GFLOPS)
apexcore gpu stress -d 0 --duration 60   # термотест 60 с → PASS/WARN/FAIL
```

Если OpenCL/GPU в системе нет — все подкоманды деградируют без ошибок и печатают
понятное сообщение; остальной apexcore работает как прежде.
